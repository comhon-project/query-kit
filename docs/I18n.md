# Internationalization

`query-kit` manages internationalization and supports the following languages:
* english (en)
* chinese (zh)
* hindi (hi)
* spanish (es)
* arabic (ar)
* french (fr)
* bengali (bn)
* portuguese (pt)
* russian (ru)
* japanese (ja)
* german (de)

## Configuration

You can define your default and fallback locales from the plugin config:

```js
import { plugin } from "@query-kit/vue";

app.use(plugin, {
  defaultLocale: 'en',
  fallbackLocale: 'en',
  // ...
})
```

## Runtime locale switching

You can change the current locale at any time using the exported `locale` ref:

```js
<script setup>
import { locale } from "@query-kit/vue";
</script>

<template>
  <button @click="() => (locale = 'fr')"> french </button>
  <button @click="() => (locale = 'en')"> english </button>
  <button @click="() => (locale = 'es')"> spanish </button>
</template>
```

The UI will reactively update to the new locale.

## Entity and enum translations

In addition to the built-in UI translations, you can provide localized labels for your own schemas:

If no translation is found, the `name` attribute from the schema definition is used as fallback. If neither is defined, the `id` is displayed.

### Entity translations

The entity translations object can contain:

- `name`: translation for the entity name
- `properties`: translations for entity properties (key is the property id)
- `scopes`: translations for scope names (key is the scope id)
- `parameters`: translations for scope parameters (key is scope id, value is an object where key is parameter id)
- `entities`: translations for local entities (key is the entity name, value follows the same structure)

```js
{
  name: 'User',
  properties: {
    first_name: 'first name',
    last_name: 'last name',
    age: 'age',
    metadata: 'metadata',
  },
  scopes: {
    active_users: 'active users',
  },
  parameters: {
    string_scope: {
      value: 'value to search',
    },
  },
  entities: {
    metadata: {
      properties: {
        label: 'label',
      },
    },
  },
}
```

### Enum translations

The enum translations object is a simple map of case id to label.

```js
{
  active: 'Active',
  inactive: 'Inactive',
  pending: 'Pending',
}
```

## Exported translation helpers

`query-kit` also exports functions to programmatically access entity translations:

```js
import {
  getEntitySchema,
  getEntityTranslation,
  getPropertyTranslation,
  getScopeTranslation,
  getScopeParameterTranslation,
} from "@query-kit/vue";
```

These are useful if you need to display entity-related labels in your own components outside of query-kit.
